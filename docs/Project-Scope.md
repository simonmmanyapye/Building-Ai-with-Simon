# Project Scope

This document defines what is included and excluded in the development of *Building AI with Simon*, a personal developer journey website documenting learning, projects, and growth into software development.

---

## In Scope

### 1. Core Website Pages & Structure

#### Home Page
- Hero section introducing “Building AI with Simon”
- Clear statement of project purpose and direction
- Navigation links to Projects and Learning Journey
- Simple call-to-action guiding user exploration

#### Vision & Mission Page/Section
- Explanation of long-term vision (learning → building → documenting AI journey)
- Mission statement focused on growth, learning, and creation
- Structured narrative layout for readability

#### About Page
- Personal introduction focused on developer journey
- Background story of how the learning process started
- Current learning focus (HTML, CSS, foundational development)
- Optional mention of tools and technologies being learned

#### Projects Page
- Static list of projects
- Each project includes:
- Title
- Short description
- Technology stack used
- GitHub repository link
- Project status (planned, in progress, completed)

#### Learning Journey Timeline
- Chronological log of learning progress
- Date-based entries
- Each entry includes:
- What was learned
- What was built or practiced
- Key insights or challenges

#### Skills Section
- Categorised skill groups:
- Frontend (HTML, CSS)
- In progress (JavaScript, Git, tooling basics)
- Future skills (React, AI concepts, advanced development)
- Simple visual representation of progression (static indicators)

#### Contact Page
- Static contact information (no backend forms)
- Links to GitHub and social platforms
- Optional email contact link

---

### 2. Design & UX Requirements

- Fully responsive design (mobile, tablet, desktop)
- Mobile-first development approach
- Documentation-style user interface:
- Clean layout
- Strong typography hierarchy
- High readability over decorative visuals

#### Design System
- Font: Montserrat
- Colour palette:
- Background: White
- Primary text: Black
- Accent colour: #220647
- Consistent spacing and layout rules
- Reusable UI patterns for sections

---

### 3. Technical Implementation

- Built using:
- HTML5 (semantic structure)
- CSS3 (layout and styling)
- Vanilla JavaScript (minimal, only where necessary)

#### Architecture
- Fully static website
- No backend services
- No database integration
- Suitable for GitHub Pages deployment

#### GitHub Integration
- All project work tracked using Git
- Repository links included in Projects page
- Standard commit workflow used for updates

---

#### 4. Content System

- All content is manually written and maintained
- Documentation-style writing approach:
- Structured and clear formatting
- Focus on clarity over marketing tone
- Markdown planning documents inform website content

---

## Out of Scope

### 1. Authentication & User Systems
- No login or signup functionality
- No user accounts or profiles
- No session management

---

### 2. Backend Systems
- No server-side processing
- No APIs
- No databases
- No admin dashboard or CMS

---

### 3. Dynamic Features
- No blog publishing system
- No comments, likes, or community interaction
- No real-time chat or messaging
- No AI chatbot integration

---

### 4. Advanced Infrastructure
- No microservices architecture
- No cloud backend services
- No multi-environment deployment complexity

---

### 5. Internationalisation
- No multi-language support
- Single language only (English)

---

### 6. Complex Application Features
- No e-commerce functionality
- No analytics dashboards
- No user-generated content systems
- No plugin or extension ecosystem
