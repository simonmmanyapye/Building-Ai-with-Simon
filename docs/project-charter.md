# Project Charter-Building Ai with Simon

---

## 1. Project Name

**Building Ai with Simon**

---

##  2. Project Overview

Building Ai with Simon is a personal software development portfolio website that documents the evolution of a beginner developer into a professional software engineer.

It is not just a portfolio of finished work, but a **living record of learning, building, and growth**.

The project will evolve over time as new skills, projects, and technical capabilities are developed.

---

##  3. Vision

To build AI systems that serve humanity by promoting truth, virtue, wisdom, and human flourishing.

This website exists as the foundational record of the journey toward that goal.

---

##  4. Mission

To continuously learn, design, and build software while documenting the entire process in public, creating meaningful projects, and developing the discipline, knowledge, and experience required to build ethical and beneficial AI systems.

---

##  5. Objectives

The project aims to:

- Establish a professional developer presence online
- Document learning and development progress over time
- Showcase technical projects and problem-solving ability
- Communicate skills, tools, and technologies clearly
- Build credibility through transparency of the learning process
- Provide a central hub for professional contact and collaboration

---

##  6. Scope (Version 1)

### In Scope

- Home page with hero section
- Vision and Mission section
- About page
- Projects page
- Learning Journey timeline
- Skills section
- Contact page
- Responsive design (mobile, tablet, desktop)
- Documentation-style UI
- GitHub integration (links to repositories)

### Out of Scope

- Authentication systems
- Blog CMS or dynamic content management
- User accounts
- Comments or community features
- AI chatbot integration
- Admin dashboard
- Complex backend infrastructure
- Multi-language support

---

##  7. Target Users

- Recruiters evaluating technical ability
- Software developers reviewing projects
- Potential collaborators
- Tech communities following the journey
- Future employers
- Self-reflection and personal tracking

---

##  8. Success Criteria

The project is considered successful when:

- The website is fully functional and deployed
- It clearly communicates vision, mission, and identity
- Projects and skills are clearly presented
- The site is responsive and accessible
- The structure reflects professional engineering practices
- Documentation is complete and version-controlled in GitHub
- The site can be extended without redesigning core structure

---

##  9. Guiding Principles

- Simplicity over complexity
- Clarity over decoration
- Consistency in design and structure
- Transparency in learning and progress
- Engineering discipline from day one
- Long-term scalability of both code and content

---

##  10. Assumptions

- The project will be built using HTML, CSS, and JavaScript
- GitHub will serve as the version control system
- GitHub Pages (or similar) will be used for deployment
- The developer is actively learning and iterating during development

---

##  11. Constraints

- Beginner-level technical experience
- Solo development
- Must remain simple enough to maintain while learning
- Must prioritize clarity over feature complexity

---

##  12. Stakeholders

- Primary: Simon M Manyapye (Developer / Owner)
- Secondary: Website visitors (recruiters, developers, collaborators)

---

##  13. Future Evolution

The project is expected to evolve into:

- A more advanced developer portfolio
- A learning and writing platform
- A foundation for future AI-related projects
- A long-term personal engineering knowledge base

---

##  14. Approval

This charter defines the foundational direction of the project and guides all design, development, and engineering decisions.

All changes to scope or direction should be reflected in updated versions of this document.
